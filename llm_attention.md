## 一、问题：为什么需要 Attention

序列处理的核心问题：**一个 token 怎么知道该参考哪些其他 token？**

之前的 RNN 逐个处理序列：

```
"I" → "love" → "cats" → "are" → "cute"
```

处理 "cute" 时，"I" 的信息要经过 4 步传递，距离远了信息会衰减丢失。

Attention 的回答：**不传递了，每个 token 直接跟所有 token 算相关性，按相关性加权取信息。**

---

## 二、输入：Embedding

每个 token 先通过 embedding 表查到一个向量。真实模型是 512 维，这里简化成 4 维：

```
x("I")    = [0.2, 0.1, 0.8, 0.3]
x("love") = [0.5, 0.9, 0.1, 0.4]
x("cats") = [0.9, 0.3, 0.7, 0.6]
```

这是每个词的**通用表示**，embedding 表也是训练学出来的。

---

## 三、三个角色：Q、K、V

Attention 需要每个 token 同时扮演三个角色：

| 角色 | 含义 | 类比 |
|------|------|------|
| **Query (Q)** | 我在找什么？ | 数据库的 SELECT WHERE |
| **Key (K)** | 我能被怎么匹配？ | 数据库的索引 |
| **Value (V)** | 匹配到我后能拿到什么？ | 数据库返回的行数据 |

一个向量干不了三件事。所以用**三个不同的权重矩阵**，把同一个 embedding 投影成三个不同的向量：

$$Q = x \times W_Q \qquad K = x \times W_K \qquad V = x \times W_V$$

区别于数据库的精确匹配，Attention 是**模糊匹配**——所有 token 都会被看到，只是权重不同。

### W_Q / W_K / W_V 是什么？

- 形状：[d_model, d_k]，比如 [512, 64]
- 随机初始化，通过反向传播训练出来
- 训练完就**固定**了，是模型的**参数**
- 它们是三个不同的"投影镜头"，从同一个 embedding 提取不同侧面的信息

### Q / K / V 是什么？

- **不是参数**，是每次输入实时算出来的
- 同一个模型，输入不同的词，Q/K/V 就不同
- 概念上是**向量**（每个 token 一个），把所有 token 的摞起来就是**矩阵**

> 💡 W 像一副固定的眼镜（训练时配好度数），Q/K/V 是你戴上眼镜后看到的东西——看的内容变了，看到的结果就变了，但眼镜没变。

## 四、矩阵乘法细节：x × W_Q 怎么算

x 是**行向量**（深度学习框架的约定，数据按行排）：

```
x("cats") = [0.9, 0.3, 0.7, 0.6]    形状 [1, 4]

W_Q = [[w₁₁  w₁₂  w₁₃  w₁₄],
       [w₂₁  w₂₂  w₂₃  w₂₄],
       [w₃₁  w₃₂  w₃₃  w₃₄],       形状 [4, 4]
       [w₄₁  w₄₂  w₄₃  w₄₄]]

Q = x × W_Q = [1, 4] × [4, 4] = [1, 4]
```

逐个元素：

```
Q[0] = 0.9×w₁₁ + 0.3×w₂₁ + 0.7×w₃₁ + 0.6×w₄₁
Q[1] = 0.9×w₁₂ + 0.3×w₂₂ + 0.7×w₃₂ + 0.6×w₄₂
Q[2] = 0.9×w₁₃ + 0.3×w₂₃ + 0.7×w₃₃ + 0.6×w₄₃
Q[3] = 0.9×w₁₄ + 0.3×w₂₄ + 0.7×w₃₄ + 0.6×w₄₄
```

结果还是一个行向量。

### 整个序列一起算

把 3 个 token 的行向量**摞起来**成矩阵，一次乘法全搞定：

```
X = [x("I")   ]   形状 [3, 4]
    [x("love")]
    [x("cats")]

Q = X × W_Q = [3, 4] × [4, 4] = [3, 4]   ← 每行是一个 token 的 Q 向量
K = X × W_K = [3, 4] × [4, 4] = [3, 4]
V = X × W_V = [3, 4] × [4, 4] = [3, 4]
```

> 数学教材喜欢列向量（W × x），工程里用行向量（x × W）更方便——batch 维在前面。结果等价，只是转置的区别。

## 五、核心公式拆解

$$\text{Attention}(Q, K, V) = \text{softmax}\!\left(\frac{QK^\top}{\sqrt{d_k}}\right) \cdot V$$

四步，每步一个目的：

### ① QKᵀ — 算所有 token 对的相关性

$$[3,4] \times [4,3] = [3,3]$$

位置 (i,j) = token i 的 Q 和 token j 的 K 的**点积**。点积越大 = 两个向量方向越接近 = 越相关。

一次矩阵乘法，**所有 token 对之间的相关性全算完**。

### ② ÷ √d_k — 缩放，防止极端

d_k 大时（比如 512），点积值会很大（±30），softmax 会趋近 one-hot（几乎只看一个 token），梯度消失训不动。

除以 √d_k 把值拉回合理范围，**让 softmax 保持"软"的**。

### ③ softmax — 变成权重

对**每一行**独立做 softmax：

$$[0.365, 0.65, 0.825] \xrightarrow{\text{softmax}} [0.26, 0.34, 0.40]$$

- 把任意范围的分数 → 0~1 之间
- 每行加起来 = 1，可以当权重用
- 分数差距越大，权重越集中

### ④ × V — 按权重提取信息

$$[3,3] \times [3,4] = [3,4]$$

每个 token 的输出 = **所有 token 的 V 的加权平均**，权重就是 softmax 的结果。

**一句话读公式：用 Q 和 K 算出"该看哪里"（相关性 → 缩放 → softmax → 权重），然后从 V 里"拿东西"（加权求和）。**

> 注意公式分两个阶段：阶段1（QKᵀ/√d_k → softmax）只用 Q 和 K 算权重，**不涉及 V**；阶段2（× V）才用权重从 V 中提取内容。就像在图书馆：先看索引卡决定哪些书相关（Q×K），再翻开书取内容（×V）。

### Step 3: QKᵀ（点积算相关性）

以 "cats" 为例，Q("cats") = [1.03, 1.12, 0.78, 1.01] 跟每个 K 做点积：

```
Q("cats") · K("I")    = 1.03×0.36 + 1.12×0.56 + 0.78×0.74 + 1.01×0.36
                       = 0.37 + 0.63 + 0.58 + 0.36 = 1.94

Q("cats") · K("love") = 1.03×0.62 + 1.12×0.60 + 0.78×0.73 + 1.01×0.77
                       = 0.64 + 0.67 + 0.57 + 0.78 = 2.66

Q("cats") · K("cats") = 1.03×0.60 + 1.12×1.12 + 0.78×1.17 + 1.01×0.65
                       = 0.62 + 1.25 + 0.91 + 0.66 = 3.44
```

### Step 4: 缩放

```
d_k = 4, √d_k = 2

[1.94, 2.66, 3.44] / 2 = [0.97, 1.33, 1.72]
```

### Step 5: Softmax

```
softmax([0.97, 1.33, 1.72])
= [e^0.97, e^1.33, e^1.72] / (e^0.97 + e^1.33 + e^1.72)
= [2.64, 3.78, 5.58] / 12.00
= [0.22, 0.32, 0.46]
```

"cats" 跟自己最相关（0.46），其次是 "love"（0.32），最后是 "I"（0.22）。

### Step 6: 加权求和 V

```
output("cats") = 0.22 × V("I") + 0.32 × V("love") + 0.46 × V("cats")

= 0.22 × [0.54, 0.44, 0.67, 0.54]
+ 0.32 × [0.55, 1.00, 0.57, 0.70]
+ 0.46 × [1.03, 0.92, 0.85, 1.08]

= [0.12, 0.10, 0.15, 0.12]
+ [0.18, 0.32, 0.18, 0.22]
+ [0.47, 0.42, 0.39, 0.50]

= [0.77, 0.84, 0.72, 0.84]
```

这个输出向量就是 "cats" **融合了整句上下文信息后的新表示**。对 "I" 和 "love" 各做一遍同样的过程，每个 token 都得到一个新向量。

### 这个新向量有什么用？

它替代了原始 embedding，成为这个 token 的**上下文感知表示**：

```
原始 x("cats")     = [0.9, 0.3, 0.7, 0.6]       ← 只知道 "cats" 这个词本身
输出 output("cats") = [0.77, 0.84, 0.72, 0.84]   ← 知道 "cats" 在这句话里的含义
```

原始 embedding 不管上下文——"I love cats" 和 "cats are evil" 里的 cats embedding 一模一样。但 attention 输出不同，因为周围的词不同，加权求和的结果就不同。**这就是"上下文感知"的来源。**

在 Transformer block 里，这个向量接着往下走：

```
x("cats")
  → Attention → output [0.77, 0.84, 0.72, 0.84]   ← 这一步的输出
  → Add & LayerNorm    （残差连接 + 归一化）
  → FFN                （两层 MLP，进一步变换）
  → Add & LayerNorm
  → 成为下一层 Transformer block 的输入
```

经过 N 层这样的 block（GPT 用 12-96 层），最后一层的输出去**预测下一个 token**：

```
最后一层 output × 词表矩阵 → softmax → 概率分布
→ "cute" 概率最高 → 输出 "cute"
```

**一句话：** 这个向量是 "cats" 在当前上下文中的语义表示，带着整句话的信息，给后续层继续加工，最终用来预测/生成下一个词。

> 💡 **Attention = Embedding with Context。** 把"死的"固定 embedding 变成"活的"上下文感知向量。这也是为什么 BERT/GPT 比之前的 Word2Vec 强——Word2Vec 里 "bank" 只有一个向量，不管是"河岸"还是"银行"。Attention 让同一个词在不同上下文中产生不同的表示。

## 七、什么是参数，什么不是

| | 是否参数 | 怎么来的 | 训练时 | 推理时 |
|---|---|---|---|---|
| **W_Q, W_K, W_V** | ✅ 是 | 随机初始化 → 反向传播训练 | 每步更新 | 固定不变 |
| **Embedding 表** | ✅ 是 | 随机初始化 → 反向传播训练 | 每步更新 | 固定不变 |
| **Q, K, V** | ❌ 不是 | x × W 实时算出 | 每次输入都不同 | 每次输入都不同 |
| **Attention 权重** | ❌ 不是 | softmax(QKᵀ/√d_k) 实时算出 | 每次输入都不同 | 每次输入都不同 |

> 💡 模型训练的是 W（镜头），不是 Q/K/V（看到的画面）。镜头固定后，不同的输入自然产生不同的画面。

### Q/K/V 真的每次都不同吗？

你可能会想：如果 embedding 和 W 都固定了，同一个词的 Q/K/V 不也固定了吗？

**第 1 层确实接近固定**——但有 positional encoding：

```
实际输入 = embedding("cats") + position(位置编码)
```

同一个 "cats" 在位置 3 和位置 50，position 不同 → x 不同 → Q/K/V 不同。

**第 2 层起就完全不固定了**——因为输入不再是原始 embedding，而是上一层 attention 的输出（已经融合了上下文）：

```
第1层：x = embedding + position       → Q/K/V 只跟词和位置有关
第2层：x = 第1层输出（已含上下文）      → Q/K/V 跟整句话有关
第3层：x = 第2层输出（更深的上下文）    → Q/K/V 跟更复杂的语义有关
...
```

所以即使同一个词在同一个位置，**只要上下文不同，第 2 层起的 Q/K/V 就不同**。

## 八、为什么有效 / 理论依据

没有先验理论说"必须这么做"。更多是工程直觉 + 实验验证：

### 1. 信息检索的思想

Attention 本质是 soft retrieval。检索系统里 query 和 document 用不同的表示是常见做法。Q 和 K 用不同投影，就是这个思路的神经网络版。

### 2. 双线性形式（Bilinear Form）

展开 Q·K 的点积（x = token A 的 embedding，y = token B 的 embedding）：

$$\text{score} = \underbrace{(xW_Q)}_{\text{token A 的 Q}} \underbrace{(yW_K)^\top}_{\text{token B 的 K}} = x(W_QW_K^\top)y^\top$$

比如算 "cats" 对 "love" 的关注分数：x = embedding("cats")，y = embedding("love")。

这是一个**学习得到的相似度函数**。比直接算 x·y（余弦相似度）更强，因为中间的 W_Q·W_Kᵀ 让模型能学到**非对称的相关性**——A 关注 B，不代表 B 同样关注 A。

### 3. 实验驱动

Vaswani 2017 的论文没有长篇理论推导"为什么 QKV"。消融实验证明：去掉投影（直接用 embedding 做 QKV）效果变差。先有结果，理论是后来的事。

## 九、Self-Attention vs Cross-Attention

| | Q 来自 | K、V 来自 | 场景 |
|---|---|---|---|
| **Self-Attention** | 自己的序列 | 自己的序列 | 序列内部建模关系（GPT） |
| **Cross-Attention** | 序列 A（decoder） | 序列 B（encoder） | 两个序列间对齐（翻译） |

GPT 这类 decoder-only 模型**只用 self-attention**。原始 Transformer（encoder-decoder，如翻译模型）两种都用。

Cross-attention 例子：翻译 "I love cats" → "我 爱 猫"

- decoder 生成 "猫" 时，Q 来自 "猫" 的 embedding
- K 和 V 来自 encoder 对 "I/love/cats" 的编码
- "猫" 的 Q 会跟 "cats" 的 K 匹配度最高 → 从 "cats" 的 V 提取最多信息

## 十、Multi-Head Attention

一个 attention head 只能学**一种**注意力模式。Multi-head 用 h 组不同的 W_Q/W_K/W_V，各自独立算 attention，最后拼起来：

$$\text{head}_i = \text{Attention}(XW_Q^i, XW_K^i, XW_V^i)$$

$$\text{MultiHead} = \text{Concat}(\text{head}_1, \dots, \text{head}_h) \times W_O$$

原始 Transformer：d_model = 512，h = 8，每个 head 的 d_k = 512 / 8 = 64

W_O 是 **Output 投影矩阵**，形状 [h × d_k, d_model] 即 [512, 512]，也是训练学出来的参数。它的作用是把 8 个 head 各自独立算出的结果**融合**成最终输出：

```
head_1 输出: [64维]
head_2 输出: [64维]
...
head_8 输出: [64维]

Concat → [512维]（拼起来）

× W_O [512, 512] → [512维]（融合多个 head 的信息，输出最终结果）
```

注意 W_O 做的不是加权平均，而是**线性变换**——可以任意组合所有 head 的所有维度，表达力远强于简单的加权平均。

不同的 head 可能学到：

- head 1：句法关系（主语 → 谓语）
- head 2：指代关系（"it" → "the cat"）
- head 3：位置相邻关系

**总计算量跟单头基本一样**（维度拆小了），但表达力更强。

## 十一、Attention 与 Transformer 的关系

Attention 是**机制**，Transformer 是**用这个机制搭的架构**。

一个 Transformer block：

```
Input
  → Multi-Head Self-Attention   ← attention 在这里
  → Add & LayerNorm
  → Feed-Forward Network (两层 MLP)
  → Add & LayerNorm
Output
```

堆 N 层这个 block 就是完整模型（GPT 用 12-96 层不等）。

演化线：

```
RNN/LSTM（序列处理，逐个算，慢）
  ↓
RNN + Attention（2014, Bahdanau — 主干还是 RNN）
  ↓
Transformer（2017, "Attention Is All You Need" — 把 RNN 扔了，纯 attention）
  ↓
GPT / BERT / Claude / ... （全是 Transformer 变体）
```
